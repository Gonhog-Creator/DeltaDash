## Current Data Available

**Projectile Data:**

**Environmental Conditions:**

**Target Variable:**
- bfd_mm (backface deformation - trauma measurement)

## Recommended Approach: Not ANOVA

Multi-variance ANOVA is not suitable because:
- ANOVA tests for differences between groups, not continuous prediction
- Your problem is regression (predicting continuous BFD values)
- Relationships are likely non-linear and involve complex interactions

## Best Approach: Machine Learning Regression

**Gradient Boosting (XGBoost/LightGBM/CatBoost) - Recommended**
- Handles non-linear relationships well
- Captures feature interactions automatically
- Provides feature importance
- Works well with structured/tabular data
- Handles missing values
- Scales from 300 to 5000+ data points without architecture changes

**Random Forest - Good alternative**
- Robust to overfitting
- Provides feature importance
- Easy to interpret

**Neural Networks - If you have large datasets**
- Can capture complex patterns
- Requires more data

## Key Feature Engineering

**Vest-Level Features:**
- Total areal density (sum of all layers)
- Layer count
- Layer sequence/order (interaction effects)
- Fabric type distribution (primary factor)
- Energy absorption capacity (areal density × material properties)
- Front vs back side (from shot data)

**Material-Level Features (aggregated per vest):**
- Average tensile strength
- Average modulus
- Fiber type distribution
- Weave type distribution
- Coating presence

**Projectile Features:**
- Kinetic energy (0.5 * mass × velocity²)
- Momentum (mass × velocity)
- Caliber diameter
- Projectile type (encoded)
- Strain-rate indicator (velocity × material type interaction)

**Environmental Features:**
- Clay temperature (slight effect)
- Humidity
- Conditioning state
- Vest size

## ML Deployment Architecture

**How it works:**
1. Train model on your data (Python script)
2. Save trained model to file (e.g., model.pkl or model.json)
3. Load model in backend API when server starts
4. API uses model to make predictions on-demand
5. Model can be retrained as new data arrives

**Scalable System Design:**
- Use XGBoost (works well from 300 to 5000+ data points)
- Same model architecture, just retrained with more data
- Model file stored in backend (e.g., backend/models/bfd_predictor.pkl)
- API endpoint: POST /api/v1/predictions/bfd
- Input: vest configuration, projectile type, velocity
- Output: predicted BFD with confidence interval

**Retraining Process:**
- Manual trigger via admin endpoint
- Automatic retraining every X new data points
- Version control for model files
- A/B testing capability to compare old vs new models

**Dashboard Integration:**

- Show feature importance breakdown

## Industry Best Practices (from GPT Analysis)

**Physics-Informed Approach:**
- Large manufacturers use FEM/FEA simulations as core, ML as surrogate model
- Your ML model serves as surrogate for expensive physical testing
- Physics-informed features critical: kinetic energy, momentum, energy absorption capacity
- Material properties under ballistic impact behave differently than normal testing

**Key Factors Industry Models:**
- Penetration/non-penetration, BFD depth, energy transfer
- Delamination, fiber pull-out, multi-hit degradation
- Temperature/humidity aging effects
- Strain-rate behavior, viscoelastic constants

**ML Usage in Industry:**
1. Surrogate models (accelerate expensive simulations)
2. Inverse material parameter estimation
3. Optimization/generative design (Bayesian optimization, genetic algorithms)

**Why Pure ML Has Limitations:**
- Ballistic impact is brutal nonlinear physics
- Requires enormous high-quality datasets
- Tiny changes (adhesive, ceramic flaws, humidity, projectile yaw) can radically alter results
- Even sophisticated models require extensive physical validation

**Validation Requirements:**
- Continuous validation against real tests
- Track model drift over time
- Alert when performance degrades
- Flag predictions extrapolating beyond training domain
- Domain-of-applicability checks for new configurations
